Projects

OMIS 105 — Santa Clara University — Team of 5

Whatcoffee Point-of-Sale Database

Designed and implemented a relational database for a local Santa Clara coffee shop to integrate their point-of-sale system. The database connects front-of-house and back-of-house operations, enabling real-time tracking of orders, inventory, revenue, and customer data.

What We Built

  • ER model with 6 entities
  • Relational schema normalized to 3NF
  • Full data dictionary
  • SQL table creation scripts
  • Populated database with realistic sample data

SQL Queries Developed

  • Employee order volume by time range
  • Average selling price with markup calculations
  • Customer order history across date ranges
  • Multi-table JOINs with GROUP BY and HAVING

Tools & Concepts

  • MySQL / SQL Workbench
  • Entity-Relationship Modeling
  • Third Normal Form (3NF)
  • Primary & Foreign Keys
  • Database normalization
